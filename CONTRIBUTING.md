# 🤝 Contributing to TurboQuant Studio

First off, thank you for considering contributing to **TurboQuant Studio**! It's people like you that make the open-source AI ecosystem amazing.

---

## 🔓 Open Source Principles

We believe in the power of shared knowledge. This project is open-source, and we want you to feel free to fork it, modify it, and use it in your own projects. 

### THE ATTRIBUTION RULE:
If you use significant portions of this codebase or UI design in your own work, we only ask for one thing: **Give credit where it is due.**
*   Mention that it was built by **Hustla**.
*   Include a link back to this repository and reference the **Ethernine Protocol**.

## 🛠️ How Can I Contribute?

### 🛰️ Reporting Bugs
*   Check the [Issues](https://github.com/agent9ether/turbo-quant-studio/issues) to see if it's already reported.
*   If not, open a new issue. Clear titles and reproduction steps are highly appreciated.

### 🧪 Feature Requests
*   We're always looking for new quantization engines (ExLlamaV2, AWQ, etc.), UI improvements, or better telemetry. Open an issue with the [Feature] tag!

### 💻 Pull Requests
1.  **Fork** the repo and create your branch from `main`.
2.  **Install** dependencies (`npm install`).
3.  **Run** the dev environment (`npm run tauri dev`).
4.  **Test** your changes. If you're adding UI, ensure it follows the **Cyber-Technical** theme (Dark mode, cyan accents).
5.  **Submit** your PR with a clear description of what you've improved!

## 📜 Code of Conduct
*   Be respectful and construction-focused.
*   Help others learn.
*   Keep the UI clean and premium.
